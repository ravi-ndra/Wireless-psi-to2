Semiconductors & Diodes
High Priority
▼

Simple Explanation

A semiconductor (like Silicon) is a material that can either conduct or block electricity — unlike metals (always conduct) or rubber (never conducts). By adding impurities (doping), we create P-type and N-type materials. Join them → P-N Junction Diode.

Real-Life Example

The charger brick in your phone uses diodes to convert AC (from wall) to DC (for battery). This is called rectification.

Key Formulas

Diode equation: I = I₀(e^(V/VT) − 1)
Forward bias: V ≈ 0.7V (Silicon), 0.3V (Germanium)
Rectifier efficiency (full wave): η ≈ 81.2%

Important Types

  • Zener Diode — Used for voltage regulation (works in reverse bias)
  • LED — Emits light (energy gap determines color)
  • Schottky Diode — Very fast switching, low voltage drop
  • Varactor Diode — Acts as variable capacitor (used in tuners)
MCQ Approach: If question asks about "voltage regulation" → Zener. If "fast switching" → Schottky. If "light emission" → LED. Always look for the keyword clue.

Transistors & Amplifiers
Very High Priority
▼

Simple Explanation

A transistor is a switch + amplifier. It has 3 terminals: Base (B), Collector (C), Emitter (E). A tiny current at the Base controls a large current at the Collector. Two types: NPN and PNP. NPN is most common.

Real-Life Example

Every smartphone has billions of transistors. A light dimmer switch uses transistor-like components to control brightness — small input controlling large output.

3 Configurations

  • Common Emitter (CE) — Highest voltage gain, 180° phase shift. Most used in amplifiers.
  • Common Base (CB) — Best frequency response. Used in RF circuits.
  • Common Collector (CC) — Emitter follower. Voltage gain ≈ 1. High input impedance.

Key Formulas

β (current gain) = IC/IB (typically 50–200)
α = IC/IE = β/(β+1)
IC = β × IB
Voltage gain (CE) = −β × RC/re
MCQ Approach: "Phase shift of 180°" → Common Emitter. "No phase shift" → Common Base or Common Collector. "Impedance matching" → Common Collector. Memorize this table.

Regulated Power Supply & Feedback
Medium Priority
▼

Simple Explanation

A power supply converts 230V AC → stable DC. Steps: Transformer → Rectifier → Filter → Regulator. Feedback means taking output and feeding it back to the input. Negative feedback improves stability, reduces distortion.

Benefits of Negative Feedback

  • Reduces distortion and noise
  • Increases bandwidth
  • Stabilizes gain
  • Changes input/output impedance
Gain with feedback: Af = A/(1 + Aβ)
Bandwidth with feedback: BWf = BW × (1 + Aβ)
Gain-Bandwidth product = constant
MCQ Approach: If asked what negative feedback reduces → distortion + noise. What it increases → bandwidth, stability. Gain goes DOWN, bandwidth goes UP.

Op-Amps & Linear ICs
High Priority
▼

Simple Explanation

Op-Amp (Operational Amplifier) is a very high-gain voltage amplifier with 2 inputs (inverting −, non-inverting +) and 1 output. Ideal op-amp has infinite gain, infinite input impedance, zero output impedance.

Common Configurations

  • Inverting Amplifier: Gain = −Rf/Rin (negative = phase inversion)
  • Non-Inverting Amplifier: Gain = 1 + Rf/Rin
  • Voltage Follower: Gain = 1 (buffer)
  • Summing Amplifier: Adds multiple voltages
  • Differentiator / Integrator: Uses capacitors
Inverting: Vout = −(Rf/Rin) × Vin
Non-inverting: Vout = (1 + Rf/Rin) × Vin
CMRR = differential gain / common-mode gain (higher = better)
MCQ Approach: Op-Amp questions are formula-based. Remember: Inverting = negative gain. Non-inverting = +1 added to ratio. Virtual ground concept: both inputs are at same potential in ideal op-amp.

AM, FM, PM Modulation
Very High Priority
▼

Simple Explanation

Modulation = mixing a low-frequency message signal with a high-frequency carrier to transmit over long distances. Think of it as putting a letter (message) inside an envelope (carrier) to mail it.

Three Types

  • AM (Amplitude Modulation) — Carrier amplitude changes. Simple, but noisy. Used in medium wave radio.
  • FM (Frequency Modulation) — Carrier frequency changes. Better quality, less noise. Used in FM radio (88–108 MHz).
  • PM (Phase Modulation) — Carrier phase changes. Used in digital data. Related to FM mathematically.

Key Formulas

AM: ma = (Amax−Amin)/(Amax+Amin) [modulation index, 0 to 1]
AM bandwidth: BW = 2fm
FM: mf = Δf/fm [deviation ratio]
FM bandwidth (Carson's Rule): BW ≈ 2(Δf + fm)
AM power: Pt = Pc(1 + ma²/2)
MCQ Approach: "Noise immunity" → FM wins. "Power efficiency" → SSB wins. "Simplest receiver" → AM. Bandwidth questions: AM = 2×signal freq; FM = use Carson's rule. Always calculate step by step.

Mobile Communication (GSM, CDMA)
High Priority
▼

Simple Explanation

Cellular networks divide geography into "cells." Each cell has a base station. When you move, your call "hands off" to the next cell. GSM uses TDMA (time slots). CDMA uses code division — everyone talks simultaneously but with different codes.

Key Concepts

  • GSM frequency: 900 MHz / 1800 MHz bands
  • CDMA advantage: Better capacity, soft handoff
  • 3G → 4G LTE → 5G: Each generation increases data speed
  • OFDM: Multi-carrier technique used in 4G/5G
  • Handoff/Handover: Moving between base stations
MCQ Approach: GSM = TDMA. CDMA = Code Division. 4G = LTE = OFDM. 5G = millimeter wave + massive MIMO. These associations are tested directly.

Fibre Optics & Satellite Communication
Medium Priority
▼

Simple Explanation

Fibre optics transmit data as light through glass/plastic fibres using Total Internal Reflection (TIR). Much faster than copper wires. Satellites orbit Earth and relay signals — used for TV, GPS, weather forecasting.

Fibre Optic Key Points

  • Total Internal Reflection: Light bounces inside fibre when angle > critical angle
  • Single mode: One path, less dispersion, long distance
  • Multi mode: Many paths, shorter distance
  • Attenuation: Signal loss per km (dB/km)

Satellite Orbits

  • GEO: 36,000 km, geostationary (TV broadcast)
  • LEO: 200–1200 km, low delay (Starlink)
  • MEO: 1200–35,000 km (GPS)
Critical angle: θc = sin⁻¹(n2/n1)
Numerical Aperture: NA = √(n1²−n2²)
MCQ Approach: TIR is why fibre optics work. Single mode = longer distance. GEO satellite = 36,000 km = no movement. GPS uses MEO orbit.

Number Systems & Logic Gates
Very High Priority
▼

Number Systems

  • Binary (Base 2): Only 0 and 1 — computers use this
  • Octal (Base 8): Digits 0–7
  • Hexadecimal (Base 16): 0–9 and A–F
Decimal → Binary: Divide by 2, write remainders bottom-up
Binary → Hex: Group 4 bits at a time
1s Complement: Flip all bits
2s Complement: Flip all bits + add 1 (used for subtraction)

Logic Gates — Truth Tables

  • AND: Output 1 only if ALL inputs are 1
  • OR: Output 1 if ANY input is 1
  • NOT: Inverts the input
  • NAND: AND + NOT. Universal gate — can make any other gate
  • NOR: OR + NOT. Also universal gate
  • XOR: Output 1 if inputs are DIFFERENT
MCQ Approach: "Universal gate" = NAND or NOR. XOR = parity checker. If all inputs same → XOR output is 0. De Morgan's theorem: NOT(A·B) = NOT A + NOT B

Flip-Flops, Counters & Registers
High Priority
▼

Simple Explanation

A flip-flop is a memory cell — it stores 1 bit (0 or 1). They're triggered by a clock signal. Chain flip-flops together → Counter or Register.

Types of Flip-Flops

  • SR Flip-Flop: Set-Reset. Forbidden state: S=R=1
  • JK Flip-Flop: Fixes SR forbidden state. J=K=1 → Toggle
  • D Flip-Flop: Data/Delay. Q follows D on clock edge. Most used.
  • T Flip-Flop: Toggle. Used in counters.
n-bit counter: counts from 0 to 2ⁿ−1
Modulo-N counter: resets after count N
3 flip-flops → 3-bit counter → counts 0 to 7
MCQ Approach: "Forbidden state" → SR flip-flop. "Toggle" → T or JK (J=K=1). "Most widely used" → D flip-flop. Count states = 2ⁿ where n = number of flip-flops.

8085 Microprocessor
High Priority
▼

Simple Explanation

The 8085 is an 8-bit microprocessor by Intel (1977). It processes 8 bits at a time. It has registers, an ALU (Arithmetic Logic Unit), and control unit. It follows the fetch-decode-execute cycle.

Key Facts

  • 8-bit data bus, 16-bit address bus → can address 2¹⁶ = 64KB memory
  • Clock speed: up to 3 MHz
  • Registers: A (accumulator), B, C, D, E, H, L, SP, PC
  • Flag register: S, Z, AC, P, CY

Common Instructions

  • MOV A, B — Move B to A
  • ADD B — Add B to accumulator
  • MVI A, 05H — Load immediate value into A
  • JMP address — Jump to address
  • CALL / RET — Subroutine call and return
MCQ Approach: 8085 = 8-bit. 64KB address space. Accumulator = A register (most operations happen here). Flags: Zero flag (Z) set when result is 0. Carry flag (CY) set on overflow.

Computer Networks & TCP/IP
High Priority
▼

OSI Model (7 Layers) — Must Memorize!

Mnemonic: "Please Do Not Throw Sausage Pizza Away"

  • 7 — Application (HTTP, FTP, SMTP)
  • 6 — Presentation (Encryption, Compression)
  • 5 — Session (Start/Stop connections)
  • 4 — Transport (TCP/UDP, port numbers)
  • 3 — Network (IP routing)
  • 2 — Data Link (MAC address, frames)
  • 1 — Physical (Cables, signals)

TCP vs UDP

  • TCP: Reliable, connection-oriented, slower. Used for web, email.
  • UDP: Fast, connectionless, no guarantee. Used for video call, gaming.
IP classes: A (0-127), B (128-191), C (192-223)
Private IP: 192.168.x.x, 10.x.x.x, 172.16-31.x.x
Subnet mask Class C: 255.255.255.0
MCQ Approach: OSI layer questions: "routing" = Network (Layer 3). "MAC address" = Data Link (Layer 2). "HTTP" = Application (Layer 7). TCP = reliable. UDP = fast. Memorize these instantly.

Network Security
Medium Priority
▼

Key Concepts

  • Symmetric Encryption: Same key for encrypt + decrypt. Fast. Examples: AES, DES.
  • Asymmetric Encryption: Public key encrypts, Private key decrypts. Slower but secure. RSA.
  • Hashing: One-way function. MD5, SHA-256. Used for passwords.
  • Digital Signature: Proves identity + data integrity.
  • Firewall: Filters traffic based on rules.
  • SSL/TLS: Secure web communication (HTTPS).
MCQ Approach: AES/DES = Symmetric. RSA = Asymmetric. "One-way" = Hashing. "Same key both sides" = Symmetric. "Public/private key pair" = Asymmetric/PKI.

Math & Reasoning Shortcuts ⚡

⏱️ Time & Work Formula Trick

If A can do a job in a days and B in b days, together they finish in:

Together = (a × b) / (a + b) days

If 3 people: 1/Total = 1/A + 1/B + 1/C

Q: A finishes in 10 days, B in 15 days. Together?
✅ (10×15)/(10+15) = 150/25 = 6 days
🚗 Speed, Time & Distance Triangle Method
Speed = Distance / Time
Distance = Speed × Time
Time = Distance / Speed

Unit conversion: 1 km/h = 5/18 m/s | 1 m/s = 18/5 km/h

Average Speed Trick: If same distance at speed S1 and S2:

Average speed = 2S1×S2 / (S1+S2) ← NOT (S1+S2)/2 !
Q: Travels 60km at 40 km/h, returns at 60 km/h. Average speed?
✅ 2×40×60 / (40+60) = 4800/100 = 48 km/h
📊 Percentage Multiplier Trick

Instead of calculating %, use multipliers:

+20% = ×1.2 | −20% = ×0.8
+50% = ×1.5 | −50% = ×0.5
+10% then −10% = ×1.1×0.9 = ×0.99 = net −1%

Successive Discount Trick:

Two discounts x% and y%: Net = x + y − xy/100
Q: 20% and 30% successive discount. Net discount?
✅ 20 + 30 − (20×30)/100 = 50 − 6 = 44%
⚖️ Ratio & Proportion Quick Method
If a:b = m:n, then a = mk, b = nk for some k
Total = a+b = (m+n)k
Q: Ratio of A:B = 3:5. Total = 80. Find A and B.
✅ Total parts = 3+5 = 8. Each part = 80/8 = 10.
A = 3×10 = 30, B = 5×10 = 50
💰 Simple vs Compound Interest Formula
SI = P × R × T / 100
CI = P × (1 + R/100)^T − P
CI−SI (2 years) = P × (R/100)²
Q: P=1000, R=10%, T=2 years. Find CI−SI.
✅ CI−SI = 1000 × (0.1)² = 1000 × 0.01 = ₹10
🧩 Reasoning Patterns Pattern Recognition

Common patterns to memorize:

  • Number Series: Look for +/− constant, ×/÷, squares, alternating patterns
  • Coding-Decoding: Check if letters shift by fixed number (A→D = +3)
  • Direction: Draw the path. Final = straight-line distance. Use Pythagoras.
  • Blood Relations: Make a family tree diagram
  • Syllogism: Use Venn diagrams. "All A are B" = A circle inside B circle.
Series: 2, 6, 12, 20, 30, ?
✅ Differences: 4, 6, 8, 10, 12 → Next = 30+12 = 42

Exam Strategy & Smart Tips 🧠

🎯 How to Prepare in Limited Time

1
Focus on HIGH-YIELD topics first. Part B is 120 marks. Electronics (transistors, op-amps, modulation) + Digital (gates, flip-flops) + Networks = ~80% of Part B. Start there, not from topic 1 linearly.
2
Learn concepts, then immediately practice MCQs. Don't read 3 hours then practice. Learn for 45 min, solve 15–20 MCQs right away. This is called "interleaved practice" and it works 3× better.
3
Make a "formula/concept card" for each topic. A4 paper, key formulas only. By exam day you'll have ~15 sheets. Review them every weekend — this replaces re-reading 100 pages.
4
Use the "E" option strategically. The exam has -0.25 for wrong answers but 0 for "Not Attempted." If you're less than 60% sure, choose E. If you're 70%+ sure, attempt it. The math favors attempting when you can eliminate 1–2 options.
5
Phone screen time → Anki flashcards. Install Anki (free). Create flashcards for formulas. Spend 10 min on phone = 10 min revising. Same time, completely different outcome.

🎲 Smart MCQ Guessing Strategy

✅ When to Attempt
  • You can eliminate 2+ wrong options
  • You recognize the topic but unsure of exact answer
  • Two options seem similar — pick the more specific one
  • Your gut feeling after studying the topic
❌ When to Use "E" (Skip)
  • You have no idea about the topic at all
  • All 4 options look equally plausible
  • Numerical calculation will take too long
  • Never seen this concept during preparation

⚠️ Common Mistakes to Avoid

✗
Reading without practicing MCQs. Reading creates an illusion of understanding. Only MCQ practice reveals real gaps. After every 45-min study session: solve 15 questions.
✗
Ignoring Part A (80 marks = 40% of exam!). Part A is reasoning + aptitude + constitution. These are scoreable with just 2 weeks of focused practice. Don't skip it thinking "it's easy."
✗
Studying new topics in the last 3 days. New info before exam = confusion + panic. Last 3 days = only revision of what you already know.
✗
Trying to cover everything equally. You have 45 days and 11 topics. Prioritize: Electronics (30% time) → Communication (25%) → Digital (15%) → Computer (15%) → Part A (15%).
